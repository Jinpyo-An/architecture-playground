# 프롬프트 엔지니어링

프롬프트 엔지니어링은 AI에게 어떤 역할로, 어떤 목표를 가지고, 어떤 제약 안에서, 어떤 형식으로 답할지 정하는 작업이다.

핵심은 모델이 자유롭게 답하게 두는 것이 아니라, 원하는 결과가 나오도록 입력 지시를 설계하는 데 있다.

## 무엇을 정하는가

- 역할: 모델이 어떤 입장으로 답할지
- 목표: 무엇을 해야 하는지
- 제약: 무엇을 하면 안 되는지
- 출력 형식: 어떤 구조로 답할지

## 모델별로 달라지는 기준

- 텍스트 LLM: 역할 / 목표 / 제약 / 출력형식
- 이미지 모델: 스타일 / 구도 / 조명 / 색감
- 영상 모델: 장면 / 카메라 / 움직임 / 길이 / 화면비
- 음악(Suno): 장르 / 무드 / BPM / 보컬 / 곡 구조 / 가사

같은 프롬프트 엔지니어링이라도 모델이 해석하는 신호가 다르기 때문에, 강조해야 할 항목도 달라진다.

## 왜 필요한가

- 출력이 흔들리는 것을 줄인다
- 불필요한 설명을 줄인다
- 후처리를 쉽게 만든다
- 원하는 형식의 답을 안정적으로 얻기 쉽다

## 한 줄 정의

프롬프트 엔지니어링은 AI의 답변 방식과 출력 형식을 의도대로 유도하는 입력 설계다.

## 예시

```text
당신은 상품 추천 도우미다.
아래 상품 목록만 사용해서 답하라.
없는 정보는 만들지 말아라.

질문: 영상 편집용 노트북 추천해줘
상품 목록: 맥북 프로 14, LG 그램 16
```

이렇게 쓰면 모델이 역할, 근거, 제약을 이해한 상태로 답하게 된다.
