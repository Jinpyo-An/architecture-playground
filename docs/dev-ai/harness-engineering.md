# AI 하네스 엔지니어링

이 문서는 AI 기능을 바로 호출하는 대신, 그 앞뒤를 감싸서 안정적으로 운영하는 방법을 정리한 문서입니다.

여기서 말하는 하네스는 단순한 래퍼가 아니라, 다음을 함께 다루는 제어층입니다.

- 입력 정리
- 프롬프트 조립
- 모델 호출
- 응답 파싱
- 결과 검증
- 실패 시 대체 흐름
- 로그와 평가 기록

## 한 줄로 보면

모델 자체보다, 모델을 안전하게 쓰는 구조를 먼저 만든다.

## 왜 필요한가

AI는 같은 입력에도 결과가 조금씩 달라질 수 있습니다.

그래서 하네스가 없으면 이런 문제가 생깁니다.

- 어떤 입력에서 실패했는지 추적이 어렵다
- 응답 형식이 흔들린다
- 잘못된 답변을 그대로 노출할 수 있다
- 비용과 호출 횟수를 제어하기 어렵다

하네스는 이런 문제를 줄이기 위해 모델 주변에 붙는 안전장치입니다.

## 하네스가 하는 일

1. 사용자의 입력을 정규화한다
2. 프롬프트나 요청 본문을 만든다
3. 모델이나 외부 API를 호출한다
4. 응답 형식을 파싱한다
5. 최소한의 검증을 수행한다
6. 실패하면 fallback으로 전환한다
7. 성공과 실패를 로그로 남긴다

## 기본 구조

```text
입력
  -> 전처리
  -> 프롬프트/요청 생성
  -> 모델 호출
  -> 응답 파싱
  -> 검증
  -> fallback 또는 최종 응답
```
