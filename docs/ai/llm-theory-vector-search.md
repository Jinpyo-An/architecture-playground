# LLM 벡터 검색

이 문서는 벡터 검색을 정리한 문서입니다.

## 한 줄로 보면

문장을 숫자 묶음으로 바꾸고,  
그 숫자가 비슷한 것끼리 찾는 방식입니다.

## 왜 쓰는가

일반 검색은 글자가 비슷해야 잘 찾습니다.

벡터 검색은 뜻이 비슷한 것도 찾기 쉽습니다.

예:

- 검색어: `영상 편집용 노트북`
- 결과: `맥북 프로 14`

문장이 완전히 같지 않아도 찾을 수 있습니다.

## 임베딩이 뭐냐

임베딩은 문장을 숫자 배열로 바꾼 값입니다.

예:

```text
"맥북 프로 14"
-> [0.12, -0.44, 0.91, ...]
```

이 숫자 자체를 사람이 읽을 필요는 없습니다.  
중요한 건 **비슷한 뜻이면 숫자도 비슷해진다**는 점입니다.

## 벡터 검색은 어떻게 동작하나

순서는 단순합니다.

1. 저장할 문장을 준비
2. 문장을 임베딩으로 변환
3. DB에 저장
4. 사용자가 검색어 입력
5. 검색어도 임베딩으로 변환
6. DB에 있는 벡터와 비교
7. 가장 가까운 결과 반환

## LLM은 어디에 쓰나

LLM은 보통 두 군데에서 같이 씁니다.

1. 임베딩 만들기
2. 검색된 결과를 바탕으로 답변 만들기

즉 흐름은 이렇게 볼 수 있습니다.

- 임베딩 모델: 검색용 숫자 만들기
- LLM: 검색 결과를 읽고 답변 만들기

## 실제 예시

상품 2개가 있다고 가정합니다.

- 상품 A: `맥북 프로 14, M3 칩셋`
- 상품 B: `러닝화, 쿠션 좋은 운동화`

사용자가:

- `영상 작업용 노트북 추천해줘`

라고 검색하면,  
벡터 검색은 상품 A가 더 가깝다고 판단할 수 있습니다.

## 장점

- 뜻이 비슷한 검색에 강함
- 긴 문장 검색도 가능
- 추천 기능에 잘 맞음

## 주의할 점

- 임베딩을 만들 때 비용이 듭니다
- 저장된 벡터가 있어야 검색할 수 있습니다
- 임베딩 모델과 검색 모델은 같은 기준을 쓰는 편이 좋습니다
